## PROBLEMAS ENCONTRADOS

### Problema 1
- **Archivo:** `app/components/CostMatrixView.tsx`
- **Descripción:** `{!embedded && ( ... )}` ocultaba cabecera con PdfShareButton
  cuando el componente se usaba dentro de tabs en `/matriz`.
- **Cómo se resolvió:** Se eliminó el condicional. Cabecera siempre visible.
- **Tiempo insumido:** ~5min

### Problema 2
- **Archivo:** `public/sw.js` (existía pero nunca registrado)
- **Descripción:** Sin `navigator.serviceWorker.register()`, Chrome Android
  no considera la app instalable como PWA — solo ofrece "Crear acceso directo".
- **Cómo se resolvió:** Creado `PwaRegister.tsx` (client component) que registra
  el SW al montar. Agregado al layout raíz. Mejorado manifest con `scope`
  y `purpose: maskable`.
- **Tiempo insumido:** ~10min

### Problema 3
- **Archivo:** `app/api/chat/route.ts`
- **Descripción:** El chat dependía de `GEMINI_API_KEY`. Vercel Hobby no permite
  agregar environment variables sin upgrade a Pro. Sin la key, devolvía datos
  crudos en vez de respuestas formateadas.
- **Cómo se resolvió:** Reescritura completa con motor local de intención que
  detecta palabras clave (precio, evento, servicio, resumen) y consulta las
  tablas correctas de Supabase. Gemini queda como mejora opcional si la key
  existe, pero no es necesaria.
- **Tiempo insumido:** ~20min
