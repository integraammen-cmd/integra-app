# Registro de Bugs — Integra

## Reglas
- Cada bug se documenta con: fecha, síntoma, causa raíz, solución, commit.
- Si un bug reaparece, se referencia la entrada original.
- Bugs de seguridad se marcan `[SEGURIDAD]` y se notifican antes de commit.

## Formato
```
### BUG-XXX — [Título breve]
- **Fecha**: YYYY-MM-DD
- **Síntoma**: qué se observó.
- **Causa**: qué lo provocó.
- **Solución**: cómo se resolvió.
- **Commit**: hash.
- **Severidad**: baja / media / alta / crítica.
```
